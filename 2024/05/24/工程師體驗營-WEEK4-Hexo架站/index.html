工程師體驗營-WEEK4-Hexo+GitHub Pages

🤷什麼是Hexo？

👉 靜態部落格框架 👈

  • 簡單快速且功能強大

    Node.js 帶來超級快的檔案產生速度，上百個檔案只需幾秒就能建立完成

  • Markdown 支援

    支援所有 GitHub Flavored Markdown 的功能，甚至能在 Hexo 使用大部份的 Octopress 外掛

  • 一鍵部署

    只需要一個指令就能把網站部署到 GitHub Pages、Heroku 或其他網站

  • 豐富的外掛

    有強大的外掛系統，可安裝外掛讓 Hexo 支援 Jade、CoffeeScript

👣Hexo架站步驟

  • 環境安裝

    | 環境安裝教學

    • Node.js

    • Git

  • 建立專案資料夾 & 初始化Git存儲庫

    | 於桌面新增 project 資料夾

  • 建立部落格

    | 網站名稱為 worksBlog

  • 開啟伺服器(本地預覽)

  • 新增一篇部落格文章 (注意標題不能有空白字元)

    | Markdown 語法

  • 輸出實際網頁

    • 進行編譯(md 檔瀏覽器看不懂)

    • 生成 public 資料夾(存放 html,css,img 等靜態檔案)

  • 連結 GitHub Pages

    | 在 GitHub Pages 上部署 Hexo

    • VSCode 綁定 GitHub 帳號

    • 帳戶 > 啟用雲端變更 > GitHub

    • 在 GitHub 開啟一個網站空間(repo)

    • 調整 _config.yml 的 url

  • 更新進度

    | 先清除快取再部署


💡設計部落格の二三事

  • 客製化部落格

    ⇨ _config.yml：修改網站配置(全域設定)

  • 新增導覽列

    • 新增 about 頁面

    • 安裝 theme 內建樣版 - landscape

    • 在 themes>landscape>_config.yml 新增導覽列及路徑

  • 加入 tags

    | 在該文章 .md 檔的表頭

  • 統一管理圖片

    圖床服務：Firebase
