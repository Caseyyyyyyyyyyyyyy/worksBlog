工程師體驗營-WEEK3-RWD

🤷什麼是RWD(Responsive Web Design)？

1. 用途

自動調整網頁的布局和內容 ⇨ 網站能夠在各種設備和屏幕尺寸上正常顯示和運作

2. 主要技術

  • 彈性網格布局

使用百分比(%)或彈性單位(em、rem)來定義網頁元素的寬高
⇨ 布局能夠根據屏幕大小自動調整

  • 彈性圖片

圖片大小以相對單位(%)設置
⇨ 能根據容器大小自動調整，而不會超出其容器範圍

  • 媒體查詢(Media Queries)

根據不同的設備特性(螢幕寬度、高度、解析度)應用不同的樣式表，以調整網頁布局和內容

3. 好處

  • 提升使用者體驗
  • 提高搜尋引擎優化(SEO)
  • 節省開發和維護成本

👣開發要點

1. 環境建立

  • viewport設定

  • CSS Media Queries 語法

媒體類型：screen(螢幕裝置)、print(列印裝置)、…
條件：max-width: 992px、max-width: 768px、…

2. 權重覆蓋遊戲

==先看權重，再看先後==

  • HTML 標籤 ⇨ 1 分
  • class 選擇器 ⇨ 10 分
  • ID 選擇器 ⇨ 100 分 (id多用來控制錨點)
  • inline style ⇨ 1000 分
  • !important ⇨ 10000 分

3. 寬度與單位

  • max-width

width: 1200px; ❌
寫死寬度。當視區(viewport)寬度更動時，會爆版(出現x軸)

  • %

% 以父層為基準

4. 排版

單欄 vs. 雙欄 vs. 三欄 ⇨ 只要改變 width 就好！

5. 斷點(breakpoints)規劃

⏹️小試身手-兩種讓元素「多欄變單欄」的方法

1. 改變主軸方向 (適用於不規則的資訊)

2. 改變寬度 (適用於規則的資訊 e.g., 產品列表)

👻KEEP IN MIND

  • 出現x軸scroll ⇨ 死刑

max-width ✅

width ❌

  • 圖片糊了 ⇨ 垃圾

圖片放大會失真(解析度變差)，
所以寧可原始圖片大一點，後續再去縮小比例
