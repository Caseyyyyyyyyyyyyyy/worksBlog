🤷什麼是Hexo？

👉 靜態部落格框架 👈

  • 簡單快速且功能強大

    Node.js 帶來超級快的檔案產生速度，上百個檔案只需幾秒就能建立完成

  • Markdown 支援

    支援所有 GitHub Flavored Markdown 的功能，甚至能在 Hexo 使用大部份的 Octopress 外掛

  • 一鍵部署

    只需要一個指令就能把網站部署到 GitHub Pages、Heroku 或其他網站

  • 豐富的外掛

    有強大的外掛系統，可安裝外掛讓 Hexo 支援 Jade、CoffeeScript

工程師體驗營-WEEK3-RWD

🤷什麼是RWD(Responsive Web Design)？

  • 用途

    自動調整網頁的布局和內容 ⇨ 網站能夠在各種設備和屏幕尺寸上正常顯示和運作

  • 主要技術

    • 彈性網格布局

      使用百分比(%)或彈性單位(em、rem)來定義網頁元素的寬高
      ⇨ 布局能夠根據屏幕大小自動調整

    • 彈性圖片

      圖片大小以相對單位(%)設置
      ⇨ 能根據容器大小自動調整，而不會超出其容器範圍

    • 媒體查詢(Media Queries)

      根據不同的設備特性(螢幕寬度、高度、解析度)應用不同的樣式表，以調整網頁布局和內容

  • 好處

    • 提升使用者體驗

    • 提高搜尋引擎優化(SEO)

    • 節省開發和維護成本


👣開發要點

  • 環境建立

    • viewport設定

    • CSS Media Queries 語法

      媒體類型：screen(螢幕裝置)、print(列印裝置)、…
      條件：max-width: 992px、max-width: 768px、…

  • 權重覆蓋遊戲

    先看權重，再看先後

    • HTML 標籤 ⇨ 1 分
    • class 選擇器 ⇨ 10 分
    • ID 選擇器 ⇨ 100 分 (id多用來控制錨點)
    • inline style ⇨ 1000 分
    • !important ⇨ 10000 分

  • 寬度與單位

    • max-width

      width: 1200px; ❌
      ⇨ 寫死寬度。當視區(viewport)寬度更動時，會爆版(出現x軸)

    • %

      % 以父層為基準

  • 排版

    單欄 vs. 雙欄 vs. 三欄 ⇨ 只要改變 width 就好！

  • 斷點(breakpoints)規劃


⏹️小試身手-兩種讓元素「多欄變單欄」的方法

  • 改變主軸方向 (適用於不規則的資訊)

  • 改變寬度 (適用於規則的資訊 e.g., 產品列表)


👻KEEP IN MIND

  • 出現x軸scroll ⇨ 死刑

    max-width ✅

    width ❌

  • 圖片糊了 ⇨ 垃圾

    圖片放大會失真(解析度變差)，所以寧可原始圖片大一點，後續再去縮小比例

工程師體驗營-WEEK2-Flexbox

🤷什麼是Flexbox(Flexible Box)？

  • 實務上常用的排版技巧

    其他還有 grid、float、table、inline-block 等 (學海無涯真令人高興啊(‾◡◝)

  • 使用軸線對齊

    #主軸 #交錯軸

  • 為適應父元素的大小和其他元素的變化，Flexbox元素會自動調整大小和位置

    在Flexbox中，區塊元素的預設寬度取決於該元素內容，而非父元素的寬度


✍️語法淺談- display: flex;

#設置此屬性在父元素(flex container)，將使得其第一層子元素(flex item) 並排呈現

  • 當設置此屬性後，該父元素的CSS屬性會具有以下預設值


⏹️小試身手-三種讓元素「水平垂直置中」的方法

  • Flexbox

  • Grid

  • 絕對定位

    position: absolute;

    => 使子元素相對父元素進行定位

    transform: translate(-50%, -50%);

    => 用於對元素進行平移
    => 水平方向左移自身寬度的50%，垂直方向上移自身高度的50%
